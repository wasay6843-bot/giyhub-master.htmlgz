BONUS

Common mistakes to avoid

These small slips cause most first-time GitHub Pages failures.

Forgot to rename the main file to index.html
Selected Private instead of Public repository
Forgot to click Save in Pages settings
Uploaded only one file instead of the full project
Chose the wrong branch when enabling Pages
FAQ

Good questions

GitHub is a website where developers store, manage, and share their code and project files.
A repository (or "repo") is a folder on GitHub that holds all the files for one project.
A commit is a saved snapshot of your changes. Each commit permanently records what you changed and when.
GitHub Pages is a free service that turns the files in your repository into a live website.
Upload your new or edited files and commit the changes — GitHub Pages rebuilds your site automatically.
